- **Q-001** [tag: backend, knowhow] [difficulty: easy]
  - **Q:** What runtime detection rule does `runtime.py` use to decide local vs. Vercel?
  - **A:** It checks `os.environ.get("VERCEL", "NOTHING") == "1"`. Anything other than the literal string `"1"` is treated as local.
  - **Source:** `01-knowhow-inventory.md#backend-runtime-config` / `yingqiangyuan_me/runtime.py`

- **Q-002** [tag: backend, deps] [difficulty: medium]
  - **Q:** Why is `boto3-dataclass[bedrock-runtime]` used instead of plain boto3?
  - **A:** It casts Bedrock's raw dict responses into typed dataclasses so IDE autocomplete works on `response.output.message.content[0].text`.
  - **Source:** `01-knowhow-inventory.md#external-dependencies`

- **Q-003** [tag: backend, knowhow] [difficulty: medium]
  - **Q:** Why does `paths.py` use `@cached_property` for the prompt content?
  - **A:** To read each prompt file once per process and reuse it on every request, reducing cold-start filesystem I/O on Vercel.
  - **Source:** `01-knowhow-inventory.md#yingqiangyuan_me/paths.py`

- **Q-004** [tag: backend, runbook] [difficulty: easy]
  - **Q:** After editing `prompts/instruction.md`, why don't the changes appear in the chat?
  - **A:** Because `paths.py` caches the file content via `@cached_property`. The fix is to restart the FastAPI dev server.
  - **Source:** `02-runbook.md#common-failures-and-fixes`

- **Q-008** [tag: backend, knowhow] [difficulty: hard]
  - **Q:** Why is the knowledge base injected as the *first conversation turn* rather than as part of the system prompt?
  - **A:** Bedrock prompt caching only caches *prefix* tokens. Putting the knowledge base as the first user turn with a `cachePoint` means subsequent turns hit the cache and pay fewer tokens.
  - **Source:** `01-knowhow-inventory.md#yingqiangyuan_me/prompts/knowledge-base.md`

  - **A:** (1) `useChat` POSTs to `/api/chat` with AI SDK v5 body and the fingerprint header; (2) Next rewrites to `127.0.0.1:8000/api/chat`; (3) FastAPI parses with `vercel-ai-sdk-mate`, converts via `ai_sdk_adapter`, calls `ChatSession.send_message` → `bedrock_runtime_client.converse`; (4) `ai_sdk_message_generator` streams SSE events back; `useChat` accumulates `text-delta`s and renders.
  - **Source:** `01-knowhow-inventory.md#architecture-overview`

- **Q-069** [tag: knowhow, security] [difficulty: hard]
  - **Q:** Name two things in the codebase today that suggest the developer planned for rate limiting but never finished it.
  - **A:** (1) The frontend wraps fetch to inject `X-Client-Fingerprint`; (2) the chat UI's toast logic explicitly looks for `429`/`Too many requests`/`Rate limit`. Neither is enforced server-side.
  - **Source:** `03-elevation-roadmap.md#projects-not-included`

- **Q-070** [tag: deps, knowhow] [difficulty: medium]
  - **Q:** What does the `func-args` `OPT` sentinel let you do that `None` doesn't?
  - **A:** It distinguishes "the caller didn't pass this argument" from "the caller passed None". `remove_optional()` then strips OPT-marked fields before the Bedrock call so they're omitted entirely from the request.
  - **Source:** `01-knowhow-inventory.md#multi_round_bedrock_runtime_chat_manager.py`

- **Q-071** [tag: convention, knowhow] [difficulty: easy]
  - **Q:** What does the project use instead of a Makefile?
  - **A:** `mise.toml` with `[tasks.*]` sections. `mise run <task>` is the equivalent of `make <task>`.
  - **Source:** `02-runbook.md#day-to-day-commands`

- **Q-072** [tag: frontend, knowhow] [difficulty: easy]
  - **Q:** What does the `CircuitBackground` component do, and where does it appear?
  - **A:** It renders an animated SVG circuit-board background, used only in the marketing layout — not on `/chat` (which deliberately has a calmer layout).
  - **Source:** `01-knowhow-inventory.md#app/_components/common/CircuitBackground.tsx`

- **Q-073** [tag: backend, knowhow] [difficulty: hard]
  - **Q:** If the prompt cache hit rate were 0%, what would change in the user experience?
  - **A:** Each chat turn would re-pay the input-token cost for the entire knowledge base (~100s of tokens). Latency would also creep up since the prefix isn't pre-encoded.
  - **Source:** `01-knowhow-inventory.md#prompts/knowledge-base.md`

- **Q-074** [tag: knowhow] [difficulty: medium]
  - **Q:** Why is the `app/(marketing)/page.tsx` route group used instead of just putting the landing page at `app/page.tsx`?
  - **A:** So marketing and chat can have *different layouts*. Putting the landing page in a route group with its own `layout.tsx` (CircuitBackground, etc.) keeps that styling out of `/chat`.
  - **Source:** `01-knowhow-inventory.md#app/(marketing)`

- **Q-075** [tag: knowhow] [difficulty: hard]
  - **Q:** Identify two pieces of evidence in the code that suggest "PynamoDB session storage" was planned but deferred.
  - **A:** (1) `pynamodb` and `pynamodb-session-manager` declared in `pyproject.toml` but unimported; (2) `Boto3Mixin` exposes a `bsm` (BotoSesManager) explicitly documented as "for PynamoDB use" but no model is defined.
  - **Source:** `03-elevation-roadmap.md#projects-not-included`
